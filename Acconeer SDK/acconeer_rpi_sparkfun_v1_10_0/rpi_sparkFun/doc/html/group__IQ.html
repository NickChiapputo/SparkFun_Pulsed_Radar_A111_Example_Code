◆ acc_service_iq_sampling_mode_enum_t

enum acc_service_iq_sampling_mode_enum_t

Sampling mode.

The sampling mode corresponds to how point averages are taken. With mode A, averaging is done directly in hardware by repeatedly sampling the same point. With mode B, the averaging is done manually with a special measurement loop. By doing so, a higher SNR is obtained in some situations.

◆ acc_service_iq_depth_lowpass_cutoff_ratio_get()

float acc_service_iq_depth_lowpass_cutoff_ratio_get ( acc_service_configuration_t  service_configuration)

Get the cutoff for the lowpass filter in the distance domain.

The cutoff for the distance domain lowpass filter is specified as the ratio between the spatial frequency cutoff and the sample frequency. A ratio close to zero damps everything except the lowest frequencies. Increasing ratios output a wider band of spatial frequencies, and a ratio of 0.5 means that the filter is deactivated.

Parameters
[in]service_configurationThe configuration to set the distance lowpass filter cutoff for
Returns
The ratio between the configured cutoff frequency and the sampling frequency

float  cutoff_ratio 
)

Set the cutoff for the lowpass filter in the distance domain.

The cutoff for the distance domain lowpass filter is specified as the ratio between the spatial frequency cutoff and the sample frequency. An input value of zero for the cutoff ratio will configure the smoothest allowed filter. A cutoff ratio of 0.5 turns the filter off.

The set of available cutoff frequencies is limited due to internal properties of the filter implementation. Call acc_service_iq_running_average_factor_get to obtain the granted cutoff frequency ratio.

Parameters
[in]service_configurationThe configuration to set the distance lowpass filter cutoff for
[in]cutoff_ratioThe ratio between the desired cutoff frequency and the sampling frequency

◆ acc_service_iq_running_average_factor_get()

float acc_service_iq_running_average_factor_get ( acc_service_configuration_t  service_configuration)

Get running average factor.

The running average factor is the factor of which the most recent sweep is weighed against previous sweeps. Valid range is between 0.0 and 1.0 where 0.0 means that no history is weighed in, i.e filtering is effectively disabled. A factor of 1.0 means that the most recent sweep has no effect on the result, which will result in that the first sweep is forever received as the result.

Parameters
[in]service_configurationThe configuration to set the running average factor for
Returns
Running average factor

◆ acc_service_iq_stepsize_get()

uint16_t acc_service_iq_stepsize_get ( acc_service_configuration_t  service_configuration)

Get the sensor stepsize.

Gets the stepsize - the number of steps taken between each data point. A stepsize of 1 samples every possible point in the range. A stepsize of 2 samples every other point, and so on.

In the IQ service, the base step length is ~0.5mm. Thus, for example setting stepsize to 4 makes the distance between two points in the measured range ~2mm.

Parameters
[in]service_configurationThe configuration to get stepsize from
Returns
stepsize The stepsize

◆ acc_service_iq_stepsize_set()

void acc_service_iq_stepsize_set ( acc_service_configuration_t  service_configuration,
uint16_t  stepsize

Set the sensor stepsize.

Sets the stepsize - the number of steps taken between each data point. A stepsize of 1 samples every possible point in the range. A stepsize of 2 samples every other point, and so on.

In the IQ service, the base step length is ~0.5mm. Thus, for example setting stepsize to 4 makes the distance between two points in the measured range ~2mm.

The IQ service supports a stepsize of 1, 2, or 4.

Parameters
[in]service_configurationThe configuration to set stepsize in
[in]stepsizeThe stepsize
